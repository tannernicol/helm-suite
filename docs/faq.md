# FAQ

## Is this a full product?
Not yet. It is a reference blueprint with patterns and scaffolding.

## Where are the apps?
Apps are private and referenced as patterns, not shipped here.

## Can I fork and extend it?
Yes. The goal is to support customization and private deployments.

## How do I add a new module?
Follow the module patterns in the architecture and update the stack map.

## Should Pip-Boy offline training be in Helm Suite docs?
Yes. It should be highlighted as a first-class capability because it ties cloud-agent workflows (Claude/Codex) to local model ownership and ongoing quality improvement.

## Should Pip-Boy offline coding agent be highlighted?
Yes. Treat it as a flagship module in Helm Suite because it demonstrates local AI utility, privacy, and reduced cloud dependency for coding tasks.
